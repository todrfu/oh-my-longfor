== Conclusion ==
RESULT: NOT WORKS (skills.paths does not load custom skill directories in OpenCode 1.2.13)

The `skills.paths` field exists in the OpenCode JSON schema 
(https://opencode.ai/config.json) as:
  "paths": { "description": "Additional paths to skill folders", "type": "array" }

However, in testing, adding paths to this config field does NOT cause OpenCode to
discover skills from those paths. The skill count remains unchanged.

== Fallback Strategy ==
Use SYMLINKS: Create symlinks from ~/.claude/skills/<skill-name> or
~/.agents/skills/<skill-name> pointing to cloned skill directories.

Symlink structure:
  ~/.claude/skills/<skill-name> -> ~/.oml/repos/<repo-name>/skills/<skill-name>/

This is confirmed to work because ~/.claude/skills/ IS scanned by OpenCode.

== Recommendation for oml ==
In lib/config.sh:
  - PRIMARY: Attempt skills.paths first (in case future OpenCode versions support it)
  - FALLBACK: Always create symlinks as a secondary mechanism
  - This ensures forward compatibility
